Prototype Assets

Screen 1

First screen the user will see. It consists of two text inputs that ask the e-mail and password of the user, below the inputs there’s a button to start the application by signing in, there’s also a cross that exits the app.

Screen 2

If the user after entering his e-mail and password it leads second screen shows the app loading.

Screen 3

When the app loads the screen shows the user the device that he already has paired to his phone in a rectangle including the device’s name and model. The rectangle also has a tip of an arrow that invites the user to click it to access the device’s routines. Below the rectangle where the first device is displayed there’s a remove button and another rectangle with a text that says Pair new device to add a new one.

Screen 4

If the user taps yes he will be shown the new device shown on the devices screen.

Screen 5

If the user taps Pair new device will lead to this screen that tells the user that the new device is being paired through Bluetooth and that it has to be on.

Screen 6

If the user taps the remove button he will be shown these menu where he is asked if he wants to remove it.

Screen 7

It displays this screen where it displays in a series of squares with symbols the routines that the device has with the possibility of adding a new custom one in the bottom right square.

Screen 8

If the user taps an enabled routine like the one displayed in this screen to turn the lights on and off displaying the knock code that was assigned to it including certain parameters in the case of other routines that will be seen later.On the side of the code there’s an edit button that allows the user to change it.Also on the top right corner of every routine there is either a tick or a cross accompanied by an arrow tip looking down that tells if it’s disabled or enabled.

Screen 9

In this screen it is displayed a routine that was disabled and is being configured as mentioned before the routine screens have the knock code displayed with the edit button on the side. There are also the aforementioned parameters in this case there are the number that will be texted and the message sent when someone arrives and knocks the code, these parameters will be entered by the user by touching on these types of areas and by using the phone’s keyboard.

Screen 10

If the user taps on edit on any of the routine screens this screen will appear with a series of lines that will show the code as it is being entered, below these lines there’s a big square with rounded edges that indicates the user that he can tap the code using the touch screen to enter the code. Below the square there are a save and a delete button, the save button keeps the code entered while delete erases one of the beats of the knock code.

Screen 11

After the user inputs the code and taps save the code is saved and displayed on the routine’s screen.

Screen 12

Here there are more screens of another routine being programed as seen in the previous three screens in a similar fashion just asking the user how many degrees does he want to increase by each knock.

Screen 13

Choices of enable and disable used on the routines.
